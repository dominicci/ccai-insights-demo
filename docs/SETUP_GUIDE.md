# CCAI Insights Configuration Guide

This guide details the manual configuration steps required in the Google Cloud CCAI Insights Console to properly visualize and analyze the synthetic data generated by this project.

## Section 1: Setting Up Custom Highlights (Phrase Matchers)

**Official Documentation**: [Set up custom highlights](https://cloud.google.com/contact-center/insights/docs/highlight)

**Goal**: Create detectors for key moments in the conversation.

### Steps

1.  Navigate to **Resources** > **Phrase matchers**.
2.  Click **Create**.
3.  Create the following 4 matchers (Use type: **Any of**):

#### 1. Standard Greeting
*   **Role**: AGENT
*   **Phrases**:
    *   `My name is`
    *   `This is`
    *   `calling from`
    *   `Thank you for calling`

#### 2. Empathy
*   **Role**: AGENT
*   **Phrases**:
    *   `I understand`
    *   `I'm sorry to hear that`
    *   `I apologize`
    *   `I hear you`

#### 3. Save Attempt
*   **Role**: AGENT
*   **Phrases**:
    *   `better offer`
    *   `retention team`
    *   `potential savings`
    *   `solutions`

#### 4. Professional Closing
*   **Role**: AGENT
*   **Phrases**:
    *   `Have a great day`
    *   `Is there anything else`

## Section 2: Configuring the Topic Model (Manual Definition Method)

**Official Documentation**: [Topic modeling overview](https://cloud.google.com/contact-center/insights/docs/topic-model)

**Goal**: Force the model to detect specific scenarios even with small data volumes.

### Steps

1.  Navigate to **Topics** and create a new model (e.g., "Synthetic Demo V1").
2.  If the model is empty, click **+ Add Topic** to manually define them.
3.  **Crucial Step**: Enter the following names and exact descriptions (the description acts as the training phrases):

| Topic Name | Description |
| :--- | :--- |
| **Order Status Inquiry** | Customer asking for tracking number, shipping update, scheduled delivery time, or package location. |
| **Product Return Request** | Customer requesting a return label, asking about refund policy, original packaging requirements, or RMA number. |
| **Store Hours & Location** | Inquiries about opening time, closing time, weekend hours, parking availability, or store location. |
| **Password Reset Help** | Technical support for users who cant login, need a reset link, forgot password, or are locked out. |
| **Service Cancellation Retention** | Customer wants to cancel subscription or close account due to price (too expensive). Involves retention team offers. |
| **Technical Support Tier 2** | Complex technical issues requiring advanced troubleshooting or a technical specialist. |

4.  **Save** the topics.
5.  **Re-Analyze Data**: Go to the **Conversations** tab, select your uploaded files, click **Analyze**, and select this new model to apply the tags.

## Section 3: Building the Quality AI Scorecard

**Official Documentation**: [Quality AI setup guide](https://cloud.google.com/contact-center/insights/docs/scorecards)

**Goal**: Create a rubric to score agent performance.

### Steps

1.  Navigate to **Quality AI** > **Scorecards**.
2.  Create a new Scorecard named "**Core Quality Rubric**".
3.  Add the following **Yes/No** questions:

#### Question 1: "Did the agent use the standard greeting?"
*   **Instruction**: Score **Yes** if the agent explicitly introduced themselves by name or company immediately after the IVR handoff.

#### Question 2: "Did the agent demonstrate empathy?"
*   **Instruction**: Score **Yes** if the agent explicitly acknowledged the customer's distress (e.g., "I understand", "I apologize").

#### Question 3: "Did the agent use a professional closing?"
*   **Instruction**: Score **Yes** if the agent used a polite closing phrase. Score **No** if the call ended abruptly. Score **N/A** for transfers or technical drops.

4.  **Save** and apply to a conversation to test.
